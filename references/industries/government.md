# 政务数字化/智慧城市适配指南

### 典型业务场景
- 一网通办/政务服务大厅改造
- 智慧安防与城市大脑监控
- 国资云迁移与数据入湖
- 基层治理网格化管理系统

### 参与方风险预警
- **角色**：总包商。**风险**：层层转包，导致最终执行方技术能力极差。
- **角色**：监理方。**风险**：与乙方串通，虚假签收。
- **防甩锅方案**：严禁无证转包；甲方直接抽查底层开发日志；引入异地独立监理。

### 验收关键项
- **材料**：政务外网/内网的安全物理隔离证明。
- **过程管理**：资金审计专户，确保专款专用，农民工工资不拖欠。
- **交付评估**：政务系统的"群众好评率"或"办事效率提升率"量化指标。

### 落地执行蓝图 (政务数字化)

#### 1. 前期需求调研清单 (甲方自查)
- [ ] **预算来源**：确认项目资金来源（财政拨款/专项债/自筹），是否已完成预算审批流程。
- [ ] **数据资产盘点**：梳理现有政务数据资产目录，确认数据入湖/入仓的范围和标准。
- [ ] **安全等级**：确认系统涉密等级（非密/内部/秘密/机密），对应的安全防护要求。
- [ ] **跨部门协同**：确认涉及的业务部门清单，是否已取得各部门的数据共享授权。

#### 2. 服务商评标标准
- **资质权重 (30%)**：涉密信息系统集成资质、CMMI 3 级以上、政务云接入认证。
- **经验权重 (30%)**：近 3 年有 5 个以上同级别政府单位数字化项目案例，需提供验收证明。
- **信创合规 (20%)**：产品是否在政务信创目录内，是否支持国产操作系统/数据库/中间件。
- **本地化服务 (20%)**：在项目所在地是否有常驻团队，响应时效是否满足政务要求。

#### 3. 执行节点与巡检计划
- **需求确认阶段 (★核心)**：组织各部门业务骨干参与需求评审，确保功能清单覆盖所有业务场景，签字确认后锁定需求基线。
- **安全评审阶段 (★核心)**：方案必须通过网信办/保密局的安全评审，确认物理隔离/逻辑隔离方案合规。
- **联调测试阶段**：与各业务系统逐一联调，确保数据流转无断点，特别关注跨部门数据交换。
- **试运行阶段**：选取 1-2 个业务条线试运行 1 个月，收集一线人员反馈并优化。

#### 4. 合规资质办理指引
- **政府采购流程**：确认是否走公开招标/邀请招标/竞争性磋商，编制采购需求论证报告。
- **安全测评**：系统上线前需通过等保三级测评及密码应用安全性评估。
- **数据共享审批**：跨部门数据共享需经大数据管理局/政务服务局审批。
- **涉密审批**：涉及涉密信息的，需通过保密局审查，办理涉密资质。

#### 5. 应急预案 (Plan B)
- **风险 1：总包商层层转包导致质量失控** → 触发阈值：发现二级以上转包或底层执行人员资质不符。补救：甲方有权要求总包商立即整改，更换不合格分包商，逾期未整改则扣除合同额 10%-20%。
- **风险 2：监理方与乙方串通虚假签收** → 触发阈值：监理签字内容与实际验收情况不符。补救：引入异地独立监理进行复核，原监理费用不予支付，情节严重者追究法律责任。
- **风险 3：政策变动导致项目方向调整** → 触发阈值：上级部门下发新的政策文件影响项目范围。补救：合同中约定"政策变更条款"，允许甲方在政策变动时调整需求范围，乙方需配合变更且不得索赔。

#### 6. 验收与上线筹备清单
- [ ] **功能验收**：所有业务场景 100% 覆盖，群众办事流程跑通。
- [ ] **安全验收**：等保测评通过，渗透测试无高危漏洞，物理隔离合规。
- [ ] **资金审计**：资金专户流水清晰，专款专用，无挪用。
- [ ] **培训验收**：各部门操作培训完成，一线窗口人员能独立操作。
- [ ] **文档移交**：系统文档、数据字典、接口规范、运维手册、应急预案完整移交。
