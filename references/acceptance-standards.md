# 量化验收标准与手册模板

本文档指导甲方如何制定科学、可量化的验收指标，输出完整的验收手册，规避歧义。

---

## 1. 四大验收维度

| 维度 | 验收重点 | 甲方容易踩坑的歧义描述 | 建议规范表述 |
| :--- | :--- | :--- | :--- |
| **功能验收** | 业务逻辑、UI/UX、流程闭环 | "界面美观，操作流畅" | "符合 UI 设计稿（偏差<3px），点击响应延迟 < 200ms" |
| **性能验收** | 并发、响应时延、稳定性 | "支持多人同时使用" | "支持 5000 活跃用户并发，核心链路平均响应时间 < 2s" |
| **合规验收** | 安全、隐私、行业红线 | "符合安全标准" | "通过等保三级测评，敏感字段 100% 脱敏存储" |
| **售后验收** | 文档、培训、故障响应 | "提供良好售后支持" | "提供 12 个月免费维保，S1 级故障 2 小时内到场/解决" |

## 3. 验收节点设计建议

- **节点一：原型/设计确认** (支付 20%) — 确保乙方没理解错需求。
- **节点二：关键功能演示 (Demo)** (支付 30%) — 看到初步成果，降低中途跑偏风险。
- **节点三：试运行通过** (支付 30%) — 在真实场景下运行 1-2 周，无重大故障。
- **节点四：终验与移交** (支付 10%) — 资料收齐，全流程跑通。
- **节点五：质保期结束** (支付 10%) — 12 个月后，系统运行稳定。

---

## 4. 歧义条款规避指南

1.  **禁止使用模糊形容词**：如“尽快”、“合理”、“友好”、“稳定”。
2.  **明确计算公式**：如“成功率 = 成功执行次数 / 总尝试次数”。
3.  **指定测试环境**：如“在 Chrome 最新版本及 iOS 15+ 环境下验收”。
4.  **明确验收参与人**：谁签字才算数？必须在合同中列出名单。
