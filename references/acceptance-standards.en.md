# Quantifiable Acceptance Standards (EN)

This document guides the Client in setting scientific, measurable metrics to avoid ambiguity.

---

## 1. Four Dimensions of Acceptance

| Dimension | Focus | Avoid This (Ambiguous) | Use This (Quantifiable) |
| :--- | :--- | :--- | :--- |
| **Functional** | Logic, UI/UX, Flow | "Beautiful UI, smooth operation" | "Matches design within <3px; click response < 200ms" |
| **Performance** | Concurrency, Latency | "Supports many users" | "Supports 5000 concurrent users; avg latency < 2s" |
| **Compliance** | Security, Privacy | "Meets safety standards" | "Passes Level 3 Protection; 100% data encryption" |
| **Support** | Docs, Training, SLA | "Good after-sales support" | "12-month warranty; S1 issues fixed within 4h" |

## 3. Avoid Ambiguity

1. **No Vague Adjectives**: Ban "fast," "reasonable," "friendly," "stable."
2. **Define Formulas**: e.g., "Success Rate = Success / Total Attempts."
3. **Specify Environment**: e.g., "Tested on Chrome v120+ and iOS 16+."
